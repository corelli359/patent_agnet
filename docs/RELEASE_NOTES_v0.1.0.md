# v0.1.0 发布说明

发布日期：2026-03-23

## 版本概述

`v0.1.0` 是仓库首次整理后的源码发布版本。这个版本的重点不是把所有细节都做到最终态，而是把专利项目从创意到申请底稿的主流程串起来，并保证代码、部署和文档具备继续迭代的基础。

## 本版包含

### 1. 项目级主流程骨架

- 以 `PatentDraft` 作为项目容器
- 支持项目创建、项目总览、版本列表、版本恢复
- 支持将检索结果挂接到项目

### 2. 六个核心能力模块

- 发明意图总结
- 技术交底书撰写
- 专利申请文件起草
- 审查模拟
- 专利修复
- 专利检索与分析

### 3. 可运行的前后端

- FastAPI 后端
- Vite + React 前端工作台
- 默认 SQLite 持久化

### 4. 本地部署能力

- 提供 Minikube 挂载式部署脚本
- 提供 `autopat` namespace 部署清单
- 不构建业务镜像，直接挂载源码目录运行

## 适用场景

本版本适合：

- 在本机验证单用户专利流程闭环
- 继续推进产品逻辑和提示词质量
- 作为后续版本的工程基础

本版本暂不主打：

- 多用户协作
- 复杂权限体系
- 生产级高可用部署

## 已知限制

- 审查模拟和修复仍以内部流程闭环为主，离“正式审查代理级”还有迭代空间
- 检索与论证深度仍有提升空间
- 前端工作台已可用，但交付体验还可继续细化
- Minikube 挂载部署首次启动较慢，backend 仍按 Pod 生命周期重建 venv

## 升级建议

后续版本建议继续推进：

1. 审查与修复质量增强
2. 检索证据链与对比论证增强
3. 前端工作台交付体验增强
4. 发布版本号与 GitHub Release 的正式化管理
